eBaoCloud for GOYO

Business Backgrounds

  1. One band can just be paired to only one APP UserID, which means GOYO APP will refuse to establish a pairing relationship if the band being paired to or had ever been paired to some other UserID.

  2. The relationship between UserID & band is fixed during pairing period no matter on which smartphone to login this user.

  3. Once the pairing relationship established, the customer is immediately granted an insurance benefit which commencement date is just the pairing start date.

  4. The commencement date will not change if pair again with the same band.

  5. The UserID can also pair to another band in unpair status which means after unpairing to the previous one.

Business Solution

eBaoCloud provides 3 APIs to support the GOYO business

  1. Propsoal submit servcie: by call this servcie GOYO app can realtime submit the proposal data to eBaoCloud and get a proprosal No. as the response.

  2. Health data service: eBaoCloud provides a service and accept the bulk health data upload

  3. Proposal data daily retrievce servcie: eBaoCloud allow GOYO & UAL to retrieve the new proposal submitted as the daily base.

By access to these 3 services, GOYO & UAL can build a loop application.

2. Retrieve the proposal data (in CSV) as a daily basis.

This API provides a GET method and accept a date (YYYY-MM-DD) as parameter. The resposne is a CSV file will all the proposal data of that day.
Open API: GET /libff/rest/goyoProposalService/generateProposalCSV/{date}

Sample request:

curl -X GET "http://ualapi.ebaocloud.life/libff/rest/goyoProposalService/generateProposalCSV/2017-06-02" -H  "accept: text/csv"

3. Health data posting

This API provides a POST method and accept a set of health data. The request body contais 2 objects: totalCount and healthDatas . The healthDatas is an array which contains a set of healthData and the totalCount indicates the array size.
Request side needs to make sure the totalCount is equals to the healthDatas array size, otherwise, error message will be sent.

Open API: POST /libff/services/rest/GOYO/healthData

Request:

healthDataRequest {
  totalCount(number) : total record number in this request,
  healthDatas(array(HealthData)) : health data array
}
HealthData {
    proposalNo(string) : proposal number, one request refer to one set of health data,
    date(string) : date of the data, YYYY-MM-DD(2015-05-22),
    calories(number, optional): calories,
    steps(number, optional): steps,
    distance(nubmer, optional): distance,
    sleep(number, optional): sleep,
    heartRate(number, optional): heartRate
}

Response:

healthDataResponse{
  status(number): status, 0,
  errorMessages(string, optional): errorMessages, [],
  message(string, optional): message
}
